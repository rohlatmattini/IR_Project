2. تدفق عملية البحث — Search Flow

1. المستخدم يكتب استعلاماً ويختار Dataset + Model من الواجهة
↓
2. API Gateway POST /api/search — يستقبل query, model, bm25_k1, fusion_method...
↓
3. Query Refinement Service (اختياري) تصحيح إملائي → مرادفات → إثراء من السجل → ترجمة عربي
↓
4. Search Service (Facade) يختار النموذج المناسب وينسّق الاسترجاع
↓
5. Retrieval Service TF-IDF | BM25 | Embedding | Hybrid Serial | Hybrid Parallel (RRF/Linear)
↓
6. Preprocessing + Indexing معالجة الاستعلام + استخدام Inverted Index / FAISS / TF-IDF Matrix
↓
7. Ranking Service ترتيب النتائج + جلب النص الأصلي من MongoDB أو JSON
↓
8. API Gateway → Web UI إرجاع JSON: results, scores, snippets, full_text
